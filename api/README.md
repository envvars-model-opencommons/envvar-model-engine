# Contract API

Static GET endpoints. No authentication, no rate limits, no server — plain files
served over HTTPS, so any language, CI job, or `curl` can consume them.

| Endpoint | Purpose |
|---|---|
| `/index.json` | Discovery: available versions and endpoint URLs. |
| `/v1/contract.schema.json` | The JSON Schema (draft 2020-12) for a contract document. |
| `/v1/example.json` | A complete, valid example document. |

```sh
BASE=https://argenv-opencommons.github.io/argenv
curl -fsSL $BASE/v1/contract.schema.json -o contract.schema.json

# validate your project's emitted document against it
python3 -c "
import json,sys,jsonschema
schema=json.load(open('contract.schema.json'))
jsonschema.Draft202012Validator(schema).validate(json.load(open(sys.argv[1])))
print('valid')" dist/myapp.contract.json
```

## Versioning

The path carries the contract major version. `v1` stays served and stable; additive
changes (new optional fields) land in place, breaking changes appear under a new
path. `/index.json` always lists what exists.

## Publishing

These files are generated and committed, then served by GitHub Pages from the `api/`
directory. They are regenerated by CI, so they cannot drift from the model.
